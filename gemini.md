# Trekker Platform - Документация Проекта

Этот документ описывает архитектуру, структуру папок, инструменты и общие подходы к разработке приложения "Trekker Platform".

## 1. Философия и Подход

**Trekker Platform** — это мультиплатформенное приложение для персонального трекинга (привычки, финансы, цели, календарь, дневник) с интегрированным ИИ-ассистентом.

- **Единая кодовая база**: Мы используем React для создания основного интерфейса, который будет работать как в десктопной версии, так и в вебе.
- **Мультиплатформенность**: Electron используется для "обертки" React-приложения в нативные исполняемые файлы для Windows и macOS. Веб-версия будет доступна через браузер.
- **Монорепозиторий**: Проект организован как монорепозиторий с использованием `npm workspaces`. Это упрощает управление общим кодом и зависимостями между разными частями приложения (ядро, десктоп, сервер).
- **Интерфейс**: Дизайн в стиле "Чистый SaaS" — современный, легкий и отзывчивый. Используется карточная система и концепция "Бесконечного холста" (Infinite Canvas). Пользователь управляет виджетами (графики, заметки, трекеры), которые можно свободно размещать на дашбордах. Визуальный стиль: мягкие тени, скругленные углы и много свободного пространства ("воздуха"). Основной акцент на UX и читаемости данных.
- **Очистка данных**: В фазе активного тестирования мы поддерживаем чистоту базы данных, удаляя артефакты от старых фич (например, заметки для "недель" и "дней" после их удаления из календаря), чтобы избежать багов и путаницы.

## 2. Инструменты и Библиотеки

- **Язык**: TypeScript
- **Монорепозиторий**: npm Workspaces
- **Форматирование кода**: Prettier
- **Линтинг**: ESLint

### Фронтенд (Ядро приложения)
- **Фреймворк**: React
- **Сборщик**: Vite
- **Стилизация**: Bootstrap, React-Bootstrap, CSS Modules/SASS
- **Иконки**: `react-icons`
- **Графики**: `recharts`
- **Работа с датами**: `dayjs`
- **Роутинг**: `react-router-dom`
- **Календарь**: `react-big-calendar`
- **Таблицы**: В процессе выбора (планируется `ag-grid-react` или `@tanstack/react-table`)

### Десктоп-версия (Local First)
- **Фреймворк**: Electron
- **Сборщик**: `electron-vite`
- **БД (Локальная)**: SQLite + Prisma (хранение данных на устройстве пользователя)

### Бэкенд (Синхронизация - Планируется)
- **Среда выполнения**: Node.js
- **Фреймворк**: Express.js
- **Работа с БД**: Prisma (PostgreSQL для синхронизации)

## 5. Архитектурная концепция: Global Widget Bar

Мы переходим от статической панели инструментов к концепции **"Global Widget Bar"**.

-   **Философия**: Toolbar — это не просто набор кнопок, а "док" или "библиотека" активных виджетов.
-   **Single Source of Truth**: Каждый виджет (Заметка, Список задач, Трекер привычки) имеет уникальный `UUID` и хранится в БД.
-   **Мульти-инстанцирование**: Один и тот же виджет может отображаться одновременно в нескольких местах:
    -   В Глобальном Баре (как иконка/мини-версия).
    -   На Холсте (как развернутая нода).
    -   В Сайдбаре (как элемент списка).
-   **Синхронизация**: Изменение виджета в любом месте (например, редактирование текста заметки на Холсте) мгновенно отражается во всех его экземплярах (в Баре, в Календаре и т.д.) благодаря подписке на UUID.
-   **Типы Виджетов**:
    -   Заметки (Notes)
    -   Списки задач (To-Do List) — *в планах*
    -   Трекеры привычек (Habit Tracker) — *в планах*
    -   Таймеры (Pomodoro) — *в планах*
    -   Мини-календари — *в планах*
